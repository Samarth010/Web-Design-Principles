Web Design Principles

  • Don't Make Me Think by Steve Krug

  • Chapter 1 : Don't Make Me Think

    1. Nothing important should ever be more than two clicks away.

    2. Be Consistent

    3. Get Rid of Question Marks(?).

    4. Everything should be Self-Evident.

  • Chapter-2 : How we Really Use the WEB

    1. User don't read pages but scans them.

    2. User always Satisfice rather than making Optimal Choices

    3. User always muddle through things.

  • Chapter-3 : Billboard Design 101

    1. Design for scanning not reading.

    2. Use Conventions for better understanding.

    3. Always make things logically and virtually relatable.

    4. Break up pages into clearly defined areas.

    5. It should be obvious that what is clickable.

    6. The text in webpage should support scanning.

  • Chapter-4 : Animal, Vegetable or Mineral?

    1. User dosen't mind extra clicks as long as they think they are on the right path.

    2. Avoid irrevelent information.

    3. Sometimes user may require assistance.

  • Chapter-5 : Omit Needless Words

    1. Get rid of half the words on each page, then get rid of half of what’s left.

    2. There shoudn't be any happy talk in website.

    3. No need for too much instructions in website.

  • Chapter-6 : Street Signs and Breadcrumbs

    1. People won’t use your Web site if they can’t find their way around it.

    2. Hierarchy always help user to find through website..

    3. Use persistent navigation.

    4. Divide page in sections and sub-sections.

    5. Name every page.

    6. Show exact location to user by using primary navigation or breadcrumbs.

    7. Always use tabs they are hard to miss and very helpful.

  • Chapter-7 : The Big Bang Theory of Web Design

    1. The homepage should explain the purpose and content of Website.

    2. Use static list instead of pull down menus.

    3. The home page should be such that it is easier to navigate and understand the website.

  • Chapter-8 : The Farmer and the Cowman Should Be Friends

    1. The programmer delivers promises made by marketing team.

    2. Don't design a website for specific user, design as per need.

    3. Don't compare working and not working with usablity testing.

  • Chapter-9 : Usability testing on 10 cents a day

    1. Few website tesing is better than no testing.

    2. Website can be tested using any person.

    3. Always keep testing videos for future review.

    4. Don't waste time to find solutions instead try something new.

    5. Fix problems such that it doesn't cost new ones.

  • Chapter-10 : Usability as a common courtesy

    1. User can leave anytime if they think they can't find what they are looking for.

    2. My way or highway dosen't apply to the us, because user will leave if thing are not going as they want.

    3. Always provide an escape in website for problem in website.
